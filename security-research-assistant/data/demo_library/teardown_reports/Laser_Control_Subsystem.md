## 1. Subsystem Overview

The AMS-500 laser subsystem consists of two primary components: the IPG Photonics YLR-1000-SM fibre laser source and the SCANLAB IntelliSCAN III galvanometer scanner head. These are controlled by a dedicated laser control unit (LCU) that sits between the main system PLC and the laser/scanner hardware. This assessment covers the LCU, the interlock safety chain, and the control interfaces.

### 2.2 Control Interface

The laser source accepts two primary control inputs:

**Analogue power control:** A 0-10V analogue signal on a BNC connector (rear panel, labelled "POWER CTRL") sets the output power linearly from 0% to 100%. At 10.00V input, the laser delivers 1000W. The analogue signal is generated by a 16-bit DAC (Analog Devices AD5683R) on the laser control unit board. Resolution is approximately 0.015W per LSB.

**Digital enable:** A 24VDC signal on a 2-pin connector (labelled "ENABLE") arms the laser emission. This signal must be TRUE simultaneously with a valid interlock chain for emission to occur. The enable signal is routed through the safety PLC (see section 4).

**RS-232 diagnostics port:** 9-pin D-sub on the rear panel (9600 baud, 8N1). Provides read-only status: internal temperature, diode current, operating hours, alarm codes. Also accepts commands for power calibration (password-protected: factory default password "IPG2020" was found printed on the internal service label).

### 2.3 Physical Observations

The laser source is a sealed unit with no user-serviceable parts. The fibre output is protected by a metal shroud clamped to the QBH connector. The fibre routing from the laser source to the scanner head passes through a 40mm flexible conduit with a 2.5m run length. The fibre is terminated at the scanner head with a process fibre connector.

The rear panel has a keyed power connector, the BNC analogue input, the digital enable connector, the RS-232 port, and a second 9-pin D-sub marked "RESERVED / CAN" (not connected in this installation).

| Axes | 2 (X, Y galvo mirrors) |

### 3.2 Control Interface (XY2-100)

The scanner is driven by the XY2-100 digital interface -- an industry-standard galvo scanner protocol. The XY2-100 link uses LVDS signalling at 2 Mbps with 20-bit position data for each axis, updated at 100 kHz. The cable from the LCU to the scanner head is a shielded multi-conductor cable (approximately 1.5m).

The XY2-100 protocol has no authentication, encryption, or integrity verification. Position commands are accepted and executed within microseconds. The protocol specification is publicly available from SCANLAB's website.

### 3.3 Physical Observations

The scanner head is mounted on a precision-machined aluminium bracket above the build chamber. The f-theta lens (focal length 420mm, field size 250mm x 250mm) is mounted below the scanner. The beam path from the scanner mirrors to the build plane passes through a protective cover glass that is replaceable for maintenance.

## 4. Laser Control Unit (LCU)

### 4.1 Hardware Description

The LCU is a custom assembly in an anodised aluminium enclosure (200mm x 150mm x 50mm), mounted inside the upper equipment bay alongside the scanner head. Board marking: "AMS-LCU-100 Rev D.1". Power input 24VDC via a 4-pin Molex connector.

Key components identified:

- **Main processor:** Texas Instruments TMS320F28379D -- dual-core C28x DSP at 200 MHz. This is a real-time DSP commonly used in motor control and laser applications. Provides hardware floating-point, 12-bit ADC, and PWM outputs.
- **FPGA:** Lattice iCE40HX4K-TQ144 -- generates the XY2-100 timing signals for the galvo scanner at the required 100 kHz update rate.
- **DAC:** Analog Devices AD5683R (16-bit, SPI) -- generates the 0-10V analogue power control signal for the laser source.
- **CAN transceiver:** Texas Instruments SN65HVD230 -- CAN bus interface to the main controller board (MCB).
- **Ethernet PHY:** Not present. The LCU communicates only via CAN bus.
- **Optocouplers:** 4x Broadcom ACPL-064L -- isolate the 24VDC digital enable and interlock signals.

### 4.2 CAN Bus Communication

The CAN bus between the MCB and the LCU runs at 500 kbps. CAN message analysis (captured using a Peak PCAN-USB adapter on test points TP4/TP5 of the MCB):

| CAN ID | Direction | DLC | Description |
|--------|-----------|-----|-------------|
| 0x100 | MCB -> LCU | 8 | Laser power setpoint (float32, watts) + scan speed (float32, mm/s) |
| 0x101 | MCB -> LCU | 4 | Scanner X position (int32, 20-bit range) |
| 0x102 | MCB -> LCU | 4 | Scanner Y position (int32, 20-bit range) |
| 0x103 | MCB -> LCU | 2 | Control flags: bit 0 = laser enable, bit 1 = scanner enable, bit 2 = gate signal |
| 0x200 | LCU -> MCB | 8 | Laser status: actual power (float32) + temperature (float32) |
| 0x201 | LCU -> MCB | 4 | Scanner actual X position (int32) |
| 0x202 | LCU -> MCB | 4 | Scanner actual Y position (int32) |
| 0x203 | LCU -> MCB | 2 | LCU status flags: bit 0 = laser ready, bit 1 = interlock OK, bit 2 = fault |

No CAN message authentication or encryption is in use. The CAN IDs and data formats were determined entirely from passive bus monitoring during a test build cycle. The 0x100 message is the critical one -- it carries the laser power setpoint as a 32-bit IEEE 754 float directly representing watts. Injecting a modified 0x100 frame on the CAN bus would directly alter laser power output.

## 5. Safety Interlock Chain

The laser safety system operates independently of the main control logic, using a dedicated safety circuit:

### 5.1 Safety PLC

A Siemens S7-1511F-1 PN (F-CPU) is installed in a separate small enclosure adjacent to the main control cabinet. This is a safety-rated PLC running a dedicated safety program (SIL 3 / PL e capable). It has its own PROFINET connection to the Hirschmann switch.

The safety PLC manages the interlock chain and controls the laser enable signal through safety relay outputs. The safety program is separate from the standard PLC program on the S7-1516.

### 5.2 Interlock Chain Components

The interlock chain is wired in series. All conditions must be satisfied for laser emission:

1. **Key switch (front panel):** Two-position maintained switch, labelled "LASER ENABLE". Key is removable in the OFF position only. Wired to F-DI channel 0 on the safety PLC.

2. **Door interlock (build chamber):** Schmersal BNS 260-11/01ZG-ST-L magnetic safety switch with unique coded actuator. Dual-channel wiring to F-DI channels 1 and 2 (cross-monitored). Opening the build chamber door during operation triggers an immediate laser shutdown.

3. **E-stop circuit:** Two E-stop buttons (one on front panel, one on the rear service area). Both are Siemens 3SU1 series, NC contacts wired dual-channel to F-DI channels 3 and 4. The E-stop circuit is monitored for cross-faults and wire-break.

4. **Emission indicator feedback:** The red "LASER EMISSION" indicator on the front panel has a feedback contact wired to F-DI channel 5. If the indicator lamp fails (open circuit), the safety PLC will prevent emission -- ensuring the operator is always warned when the laser is active.

5. **Laser source ready signal:** A 24VDC "ready" output from the IPG laser source, confirming the laser has completed its warm-up and internal self-test. Wired to F-DI channel 6.

6. **External interlock input:** A spare terminal pair on the safety PLC terminal strip (F-DI channel 7), labelled "EXT INTERLOCK". Wired to a normally-closed loop with a shorting plug installed. Intended for connection to room-level safety systems (light curtain, area scanner).

### 5.3 Safety Outputs

The safety PLC drives two safety relay outputs (F-DQ channels 0 and 1), which together control the "LASER ENABLE" signal to the IPG laser source. Both channels must be active for the enable signal to be asserted. The safety relays are Siemens 3SK1 series with forced-guided contacts and feedback monitoring.

## 6. Security-Relevant Findings

1. **CAN bus laser control is unauthenticated.** Power setpoints, scanner positions, and enable flags are transmitted as raw CAN frames with no MACs or sequence numbers. An attacker with physical access to the CAN bus (available at test points on the MCB, or by tapping the cable between MCB and LCU) could inject frames to manipulate laser power during a build, potentially causing material defects or equipment damage.

2. **Analogue power control bypass.** The 0-10V analogue power control signal from the DAC to the laser source travels through an unshielded cable (approximately 0.5m). Injecting a voltage on this line could override the DAC output, though the interlock chain would still need to be satisfied.

3. **Safety PLC is on the same PROFINET segment.** While the safety PLC runs an independent safety program, it shares the same physical network (Hirschmann switch) as the main PLC, HMI, and communication gateway. PROFINET safety (PROFIsafe) provides protocol-level authentication, but the safety PLC's PROFINET interface is accessible from the shared network.

4. **IPG RS-232 diagnostics port default password.** The factory default calibration password "IPG2020" was found on an internal service label. If the password has not been changed, it could be used to alter laser power calibration through the serial interface, making the laser output differ from the commanded setpoint.

5. **XY2-100 scanner protocol has no security.** The digital interface to the galvo scanner executes position commands without any verification. Modified scanner coordinates could cause the laser to write outside the intended build area, damaging the build platform or optics.

6. **Physical separation of safety and control.** The safety interlock chain is hardwired and independent of the software-controlled systems -- this is a proper safety architecture. However, the safety system does not and cannot detect subtle parameter manipulation (e.g., a 5% increase in laser power) that stays within normal operating bounds but degrades part quality.
